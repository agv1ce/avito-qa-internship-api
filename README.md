## Быстрый старт

### 1. Предварительные требования
Python 3.12
pip
Git

### 2. Установка
# Клонировать репозиторий
git clone <repository-url>
cd avito-qa-assignment

# Создать виртуальное окружение
python -m venv venv

# Активировать (Windows PowerShell)
.\venv\Scripts\Activate.ps1

# Установить зависимости
pip install -r requirements.txt

# Запустить все тесты
pytest

### 3. Запуск тестов
1) pytest : Запустить все тесты
2) pytest -v : Запустить с подробным выводом
3) pytest -m smoke : Только smoke-тесты
4) pytest -m negative : Только негативные сценарии
5) pytest -m regression : Полное регрессионное тестирование
6) pytest tests/test_item_api.py : Тесты из конкретного файла
7) pytest -k "create" : Запустить тесты, содержащие "create" в имени

### 4. Allure отчёт
# Запуск с генерацией Allure-результатов
pytest --alluredir=allure-results -v

# Просмотр отчёта в браузере (рекомендуется)
allure serve allure-results

### 6. Найденные баги: 
## [BUG-001] API принимает отрицательную цену объявления

Серьёзность: Critical
Приоритет: High
Статус: Open

Описание: Микросервис позволяет создавать объявления с отрицательной ценой (`price < 0`), что нарушает бизнес-логику.

## [BUG-002] API возвращает 400 вместо 404

Серьёзность: Minor
Приоритет: Medium
Статус: Open

Описание: При запросе несуществующего ресурса возвращается код ошибки валидации (400) вместо кода отсутствия ресурса (404).

## [BUG-003] Отсутствует поле sellerID в ответе

Серьёзность: Minor
Приоритет: Low
Статус: Open

Описание: В успешном ответе на создание объявления (`POST /api/1/item`) не возвращается поле `sellerID`.
