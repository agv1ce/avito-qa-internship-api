# Отчёт о дефектах API Avito

Окружение: https://qa-internship.avito.com
Дата: 05.04.2026


## [BUG-001] API принимает отрицательную цену объявления

Серьёзность: Critical
Приоритет: High
Статус: Open

### Описание
Микросервис позволяет создавать объявления с отрицательной ценой (`price < 0`), что нарушает бизнес-логику.

### Шаги воспроизведения
1. Подготовить JSON payload.
2. Установить `"price": -100`.
3. Отправить `POST https://qa-internship.avito.com/api/1/item`.

### Фактический результат
1) Status: 200 OK
2) Объявление успешно создано.

### Ожидаемый результат
1) Status: 400 Bad Request
2) Ошибка валидации: "Price must be positive".

## [BUG-002] API возвращает 400 вместо 404

Серьёзность: Minor
Приоритет: Medium
Статус: Open

### Описание
При запросе несуществующего ресурса возвращается код ошибки валидации (400) вместо кода отсутствия ресурса (404).

### Шаги воспроизведения
Отправить `GET /api/1/item/non-existent-id`.

### Фактический результат
Status: 400 Bad Request

### Ожидаемый результат
Status: 404 Not Found

## [BUG-003] Отсутствует поле sellerID в ответе

Серьёзность: Minor
Приоритет: Low
Статус: Open

### Описание
В успешном ответе на создание объявления (`POST /api/1/item`) не возвращается поле `sellerID`.

### Шаги воспроизведения
1. Создать валидное объявление.
2. Изучить тело ответа.

### Фактический результат
В ответе есть `status`, `name`, `price`, `statistics`. Поле `sellerID` отсутствует.

### Ожидаемый результат
Ответ должен содержать полную информацию о созданном объекте, включая `sellerID`.
