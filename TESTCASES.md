## 1. Позитивные сценарии

Проверка основного функционала при валидных входных данных.

### TC-001: Успешное создание объявления

Приоритет: High
Тип: Smoke

Предусловия:
API доступен.
Сгенерирован валидный payload.

Шаги воспроизведения:

1. Сформировать JSON payload со следующими полями:
   1) `sellerID`: целое число (111111–999999)
   2) `name`: уникальная строка
   3) `price`: положительное целое число
   4) `statistics`: объект с полями `viewCount`, `likes`, `contacts`

2. Отправить POST запрос:
   `POST /api/1/item`

3. Получить ответ от сервера.

Ожидаемый результат:
- Код ответа: 200 OK
- Тело ответа содержит поле `status` со значением "Сохранили объявление - <uuid>"
- Тело ответа содержит поля `name`, `price`, `statistics`, соответствующие отправленным.

### TC-002: Получение объявления по валидному ID

Приоритет: High
Тип: Smoke

Шаги воспроизведения:

1. Создать объявление через TC-001.
2. Извлечь `id` из ответа (парсинг поля status).
3. Отправить GET запрос:
   `GET /api/1/item/{id}`

Ожидаемый результат:
- Код ответа: 200 OK
- Тело ответа содержит массив с одним элементом.
- Элемент содержит тот же `id`, `name` и `price`, что и при создании.

### TC-003: Получение всех объявлений продавца

Приоритет: Medium
Тип: Regression

Шаги воспроизведения:

1. Сгенерировать `sellerID`.
2. Создать 2 объявления с этим `sellerID`.
3. Отправить GET запрос:
   `GET /api/1/{sellerID}/item`

Ожидаемый результат:
1) Код ответа: 200 OK
2) Тело ответа содержит массив из 2 элементов.
3) У всех элементов `sellerID` совпадает с запрошенным.

### TC-004: Получение статистики по существующему объявлению

Приоритет: High
Тип: Smoke

Шаги воспроизведения:

1. Создать объявление.
2. Отправить GET запрос:
   `GET /api/1/statistic/{id}`

Ожидаемый результат:
1) Код ответа: 200 OK
2) Тело ответа содержит массив.
3) Элемент массива содержит поля `likes`, `viewCount`, `contacts` (тип int).




## 2. Негативные сценарии

Проверка обработки ошибок.

### TC-101: Создание с пустым телом запроса

Приоритет: Medium  
Тип: Negative

Шаги:
1. Отправить `POST /api/1/item` с телом `{}`.

Ожидаемый результат:
1) Код ответа: 400 Bad Request
2) Тело содержит описание ошибки.

### TC-102: Создание с невалидным типом sellerID

Приоритет: Medium  
Тип: Negative 

Шаги:
1. Отправить `POST` с `"sellerID": "string"`.

Ожидаемый результат:
Код ответа: 400 Bad Request

### TC-103: Получение несуществующего объявления

Приоритет: Medium
Тип: Negative

Шаги:
1. Отправить `GET /api/1/item/non-existent-id`.

Ожидаемый результат:
Код ответа: 400 Bad Request

### TC-104: Создание без обязательных полей в statistics

Приоритет: Medium
Тип: Negative

Шаги:
1. Отправить `POST` с пустым объектом `"statistics": {}`.

Ожидаемый результат:
1) Код ответа: 400 Bad Request
2) Сообщение об ошибке: "поле contacts обязательно".

### TC-105: Получение статистики по несуществующему ID

Приоритет: Low
Тип: Negative

Шаги:
1) Сформировать GET запрос с несуществующим ID:
2) Отправить запрос
3) Получить ответ

Ожидаемый результат:
1) Код ответа: 400 Bad Request
